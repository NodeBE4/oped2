    欧盟目前正在考虑要求社交媒体公司删除违反其28个成员国法律的内容的规则。随着德国社交媒体执行法（NetzDG）于去年10月生效，该计划得以实施，当时有超过200万本地用户的社交媒体平台被要求监视和删除“明显非法的内容”，否则将面临高达 5000 万欧元的罚款；许多不同的德国法律都包含限制某些表达形式的规定。
   </p>
   <p class="graf graf--p graf-after--p" id="4a62">
    <strong class="markup--strong markup--p-strong">
     由公司解释这些法规并采取行动，在没有任何正当程序或法院事先批准的情况下影响用户。对整个欧洲的科技公司施加类似的要求可能会导致更大的混乱和失误，从而过度损害言论自由
    </strong>
    。
   </p>
   <p class="graf graf--p graf-after--p" id="fe84">
    美国也一样。在“通信规范法”第 230 条中，美国一直存在取消“安全港”保护措施的压力。如果没有这项规定，那些在试图删除被禁内容时“犯错”的公司可能要承担严重责任，法
    <strong class="markup--strong markup--p-strong">
     案明显鼓励了这些公司加强审查，而不是保护合法的表达
    </strong>
    。

        最可怕的并不是死亡。而是即便你放弃优越的生活、甚至舍弃生命安全去选择捍卫正义，结果却是因不被理解，而难以达成改变现实的目的。曾经有中国的“勇敢者”说出这样的话，“我不怕危险，怕的是社会的麻木让我不值得冒险”。但这是真的吗？《
       </strong>
       <a class="markup--anchor markup--li-anchor" data-href="https://medium.com/@iyouport/%E8%B0%83%E6%9F%A5%E6%95%B0%E6%8D%AE-%E6%96%AF%E8%AF%BA%E7%99%BB%E6%94%B9%E5%8F%98%E4%BA%86%E5%A4%9A%E5%B0%91-%E7%BB%88%E7%BB%93%E8%80%81%E5%A4%A7%E5%93%A5%E6%9C%89%E5%A4%9A%E9%9A%BE-ea44edee35d3" href="https://medium.com/@iyouport/%E8%B0%83%E6%9F%A5%E6%95%B0%E6%8D%AE-%E6%96%AF%E8%AF%BA%E7%99%BB%E6%94%B9%E5%8F%98%E4%BA%86%E5%A4%9A%E5%B0%91-%E7%BB%88%E7%BB%93%E8%80%81%E5%A4%A7%E5%93%A5%E6%9C%89%E5%A4%9A%E9%9A%BE-ea44edee35d3" rel="noopener noreferrer" target="_blank">
        <strong class="markup--strong markup--li-strong">

        </strong>

      </li>

     <p class="graf graf--p graf-after--li" id="2c9a">
      法国，德国，匈牙利和英国政府也大大加强了其情报部门的监视权力，严重削弱了旨在保护公民基本权利的重要司法监督。意大利于 2017 年11月通过了一项法律，
      <strong class="markup--strong markup--p-strong">
       要求电信运营商存储长达六年的电话和互联网数据
      </strong>
      ，尽管 2014 年欧盟法院已经裁定此类规则构成对隐私的不成比例的侵犯，显然法院一点用都没有。
     </p>
     <h3 class="graf graf--h3 graf-after--p" id="8ac5">
      世界各地都有隐私政策更新，但真的有用吗？
     </h3>
     <p class="graf graf--p graf-after--h3" id="8fd3">
      许多国家正在颁布立法，授予个人控制公共和私人实体收集、处理和共享数据的权利。自 2017 年 6 月以来，至少有 15 个国家考虑了数据保护法，至少有 35 个国家已经制定了相关法律。
     </p>
     <p class="graf graf--p graf-after--p" id="664f">
      在阿根廷、巴西和印度尼西亚提出或通过的数据保护法与 2018 年5月生效的欧盟通用数据保护条例（GDPR）非常相似。
     </p>
     <p class="graf graf--p graf-after--p" id="1287">
      GDPR 要求数据持有者获得更有意义的“同意”，提高收集数据的透明度和原因，并为用户提供下载，传输或删除其信息的方法。
      <strong class="markup--strong markup--p-strong">
       但是
      </strong>
      ，
     </p>
     <ul class="postList">
      <li class="graf graf--li graf-after--p" id="1ec3">
       <strong class="markup--strong markup--li-strong">
        延伸阅读
       </strong>
       ：《
       <a class="markup--anchor markup--li-anchor" data-href="https://medium.com/@iyouport/%E4%B8%BA%E4%BB%80%E4%B9%88%E8%AF%B4%E6%AC%A7%E6%B4%B2%E6%96%B0%E6%B3%95%E8%A7%84%E5%8F%8D%E8%80%8C%E4%BC%9A%E5%8A%A0%E5%BC%BA-facebook-%E5%92%8Cgoogle-%E7%9A%84%E9%9C%B8%E6%9D%83-795e628592d5" href="https://medium.com/@iyouport/%E4%B8%BA%E4%BB%80%E4%B9%88%E8%AF%B4%E6%AC%A7%E6%B4%B2%E6%96%B0%E6%B3%95%E8%A7%84%E5%8F%8D%E8%80%8C%E4%BC%9A%E5%8A%A0%E5%BC%BA-facebook-%E5%92%8Cgoogle-%E7%9A%84%E9%9C%B8%E6%9D%83-795e628592d5" rel="noopener noreferrer" target="_blank">

       </a>
       》
      </li>
      <li class="graf graf--li graf-after--li" id="f1d7">
       《
       <a class="markup--anchor markup--li-anchor" data-href="https://twitter.com/iyouport_news/status/1050614798562471936" href="https://twitter.com/iyouport_news/status/1050614798562471936" rel="noopener nofollow noreferrer" target="_blank">
        从跟踪脚本数量看 Google 是 #GDPR 的最大受益人
       </a>
       》
      </li>
      <li class="graf graf--li graf-after--li" id="47e3">
       《
       <a class="markup--anchor markup--li-anchor" data-href="https://twitter.com/iyouport_news/status/1049511908527681536" href="https://twitter.com/iyouport_news/status/1049511908527681536" rel="noopener nofollow noreferrer" target="_blank">
        意想不到的后果：#GDPR 反而破坏了隐私
       </a>
       》
      </li>
     </ul>
     <p class="graf graf--p graf-after--li" id="3a3f">
      在美国，首先，加州于今年 6 月通过了主要的隐私法规；然后，在9月底，特朗普政府又公布了单一国家隐私标准的官方原则；众议院民主党本月早些时候预览了他们自己的互联网“权利法案”。
     </p>
     <p class="graf graf--p graf-after--p" id="b82b">
      简而言之，全面的隐私法规可能会进入美国。鉴于如今我们的隐私权所面对的悲惨现实，理论上这应该是值得欢迎的消息。
      <strong class="markup--strong markup--p-strong">
       但是，任何新的监管数据规范都存在严重漏洞。这些法规甚至有可能适得其反
      </strong>
      — — 例如，它们巩固了已经占主导地位的大型技术公司的地位、或者未能帮助消费者实际控制我们自己产生的数据（这可是任何新法规原本应有的主要目标啊）。
     </p>
     <p class="graf graf--p graf-after--p" id="e763">
      Ozar 在加利福尼亚州经营
      <a class="markup--anchor markup--p-anchor" data-href="https://www.brentozar.com/team/" href="https://www.brentozar.com/team/" rel="noopener nofollow noreferrer" target="_blank">
       一家小型技术咨询公司
      </a>
      ，为名为
      <a class="markup--anchor markup--p-anchor" data-href="https://en.wikipedia.org/wiki/Microsoft_SQL_Server" href="https://en.wikipedia.org/wiki/Microsoft_SQL_Server" rel="noopener nofollow noreferrer" target="_blank">
       Microsoft SQL Server
      </a>
      的数据库管理系统提供培训和故障排除服务。Ozar 的公司拥有一支由四人组成的团队，而且拥有一个小型的国际客户群。直到 5 月份欧洲监管机构开始执行名为“ 通用数据保护条例” （GDPR）的隐私法，该法案可以对违规的公司收取高达全球收入 4％ 的罚款。
     </p>
     <p class="graf graf--p graf-after--p" id="8859">
      在 GDPR 开始实施的几个月前，Ozar 宣布，
      <a class="markup--anchor markup--p-anchor" data-href="https://www.brentozar.com/archive/2017/12/gdpr-stopped-selling-stuff-europe/" href="https://www.brentozar.com/archive/2017/12/gdpr-stopped-selling-stuff-europe/" rel="noopener nofollow noreferrer" target="_blank">
       用他的话说
      </a>
      ，迫使他的公司“停止向欧洲出售产品。” Ozar 写道，作为消费者，他喜欢这些规定; 但作为一个企业，他根本承担不起合规成本或错误风险。
     </p>
     <p class="graf graf--p graf-after--p" id="6888">
      这可不是 Ozar 一个人的做法。更大的国际组织，如 洛杉矶时报 和 芝加哥论坛报 — 以及
      <a class="markup--anchor markup--p-anchor" data-href="http://www.niemanlab.org/2018/08/more-than-1000-u-s-news-sites-are-still-unavailable-in-europe-two-months-after-gdpr-took-effect/" href="http://www.niemanlab.org/2018/08/more-than-1000-u-s-news-sites-are-still-unavailable-in-europe-two-months-after-gdpr-took-effect/" rel="noopener nofollow noreferrer" target="_blank">
       超过 1,000 个其他新闻媒体
      </a>
      — 全部都是仅仅阻止了任何使用欧洲 IP 地址访问其网站的用户，而不是面对 GDPR 存在的主旨。
     </p>
     <p class="graf graf--p graf-after--p" id="bd02">
      这便令试图保护公民的法规形同虚设了。公民所面对的危险并没有去除。
     </p>
     <p class="graf graf--p graf-after--p" id="79ad">
      <strong class="markup--strong markup--p-strong">
       那么我们如何才能将这些危险降到最低？
      </strong>
     </p>
     <p class="graf graf--p graf-after--p" id="c8ff">
      首先，随着监管机构越来越认真地制定新的隐私法，他们将倾向于实施通用的、基础广泛的法规，而不是在法律中规定具体的处方。即使在快速发展的技术世界中，编写一般规则也总是比更明确的建议更容易，但这些立法者应尽可能避免这种诱惑。
     </p>
     <p class="graf graf--p graf-after--p" id="dc00">
      无差别对待所有组织的过于宽泛的法规最终可能会鼓励“数据垄断” — — 只有少数公司可以利用我们的所有数据。有些组织将拥有充分的资源来遵守复杂且高度模糊的法律; 而其他人（如Ozar 和那些媒体平台等）则没有这样的资源。
     </p>
     <p class="graf graf--p graf-after--p" id="ddba">
      这意味着数据的监管负担应该分层，以便
      <strong class="markup--strong markup--p-strong">
       有差异的组织的合规成本符合其自身条件
      </strong>
      。加利福尼亚州的“消费者隐私法”通过选择特定的业务部门（如 许多小型组织）直接面对这一问题。并且，任何新法规的合规成本都不应该给世界上已经占主导地位的科技公司带来额外的好处。
     </p>
     <p class="graf graf--p graf-after--p" id="d8cf">
      其次，相关地，一些组织越来越多地负责我们的绝大部分数据，这对我们的隐私和技术创新都构成了巨大的危险。所以，任何新的隐私法规都
      <strong class="markup--strong markup--p-strong">
       必须积极地激励较小的组织
      </strong>
      共享或汇集数据，以便他们能够与更大的数据驱动型组织相竞争。
     </p>
     <p class="graf graf--p graf-after--p" id="87f5">
      解决此问题的一种可能方案是鼓励使用隐私增强技术或 PET，例如差异隐私、同态加密、联合学习（分布式机器学习）等。长期以来，私人倡导者所提倡的 PET 有助于平衡数据效用与其隐私和安全性之间的权衡。
     </p>
     <p class="graf graf--p graf-after--p" id="3bab">
      最后，
      <strong class="markup--strong markup--p-strong">
       “用户同意”
      </strong>
      （用户主动同意在特定时间点收集数据的想法）
      <strong class="markup--strong markup--p-strong">
       不再能够在保护我们的隐私方面发挥核心作用
      </strong>
      。这一直是主要隐私框架的关键方面（想想你点击进入网站的所有“我接受”按钮）。但是在大数据和机器学习的时代，我们根本无法知道我们在点击“接受”时所放弃的个人权利的真正价值。
     </p>
     <p class="graf graf--p graf-after--p" id="2034">
      机器学习的全部价值在于它能够大规模地检测模式。在任何特定时间，放弃少量数据的隐私成本都很低; 然而，随着时间的推移，成本会变得巨大。
      <a class="markup--anchor markup--p-anchor" data-href="https://www.nytimes.com/2012/02/19/magazine/shopping-habits.html?_r=1&amp;hp=&amp;pagewanted=all" href="https://www.nytimes.com/2012/02/19/magazine/shopping-habits.html?_r=1&amp;hp=&amp;pagewanted=all" rel="noopener nofollow noreferrer" target="_blank">
       一个著名的被反复引用的例子
      </a>
      就是，一个青少年在她的家人知道她怀孕之前互联网就知道了，仅仅基于她的购物习惯，这只是很多这样的例子之一。
     </p>
     <p class="graf graf--p graf-after--p" id="2d42">
      因此，我们无法假设我们完全了解在任何单一时间点放弃的隐私究竟意味着什么。
      <strong class="markup--strong markup--p-strong">
       消费者必须能够做到在数据收集后很长时间内对其数据行使权利，这些权利应包括限制数据的使用方式
      </strong>
      。
     </p>
     <p class="graf graf--p graf-after--p" id="54b7">
      除非我们的法律能够正确地适应新的数字技术 — 除非他们能够在合规负担的成本与他们寻求维护的隐私权价值之间取得平衡 — 否则，我们会冒一些非常现实的风险。我们都可以很容易地实施新的法律，而这些法律不能保护我们的隐私，同时也会阻碍新技术的使用。公民有权要求立法者规避上述问题。
     </p>
     <p class="graf graf--p graf-after--p" id="59f5">
      中国目前还没有隐私保护法案，我们需要争取将来能有，于是
      <strong class="markup--strong markup--p-strong">
       上述问题对中国公民来说非常重要：关于如何在最大程度上避免被立法者欺骗
      </strong>
      。
     </p>

     <p class="graf graf--p graf-after--figure" id="aaa4">
      <strong class="markup--strong markup--p-strong">
       任何真正的数据保护系统都应该赋予个人控制自己信息的权力，同时确保互联网无国界
      </strong>
     </p>
     <p class="graf graf--p graf-after--p" id="ec47">
      政府、私营公司和研究人员越来越渴望获取大量的个人信息，将其用于从政治压制、选举操纵、社交工程网络钓鱼、恶意监视软件、到人工智能算法开发的各种各样的目的。然而，个人通常没有多少选择来抵制这种窒息的压迫。
     </p>
     <p class="graf graf--p graf-after--p" id="cb9f">
      政府和技术公司不应强迫用户做出如此明确的选择，而应努力提高个人数据使用方式的透明度，实现平台之间的数据可移植性，并允许人们审查和删除所收集的有关他们的所有数据。
     </p>

     <p class="graf graf--p graf-after--p" id="5ebd">

       当政府机构访问公民的数据文件时，公民会得到通知
      </strong>
      。鉴于爱沙尼亚强大的隐私权法律框架，该系统提供了更大的保护。
     </p>
     <p class="graf graf--p graf-after--p" id="8a99">
      2018 年，爱沙尼亚宣布计划将 X-Road 平台扩展到境外。这一步骤源于对开放社会和国际科技公司需要确保在刚性国家框架之外保护数据的基本认识。任何真正的数据保护系统都应该赋予个人控制自己个人信息的权力，同时确保互联网保持无国界。
     </p>
     <p class="graf graf--p graf-after--p" id="543a">
      <strong class="markup--strong markup--p-strong">
       — — 建议 — —
      </strong>
     </p>
     <p class="graf graf--p graf-after--p" id="d3a4">
      其实很简单。IYP 支持 Purism 首席执行官 Todd Weaver 的五个基本原则：
     </p>
     <ul class="postList">
      <li class="graf graf--li graf-after--p" id="08d1">
       <strong class="markup--strong markup--li-strong">
        变更服务提供者的权利
       </strong>
       ：如果一个人想要更换服务提供者，他们可以轻松地转移到另一个服务（分散化服务）；
      </li>
      <li class="graf graf--li graf-after--li" id="d127">
       <strong class="markup--strong markup--li-strong">
        保护个人数据的权利
       </strong>
       ：一个人拥有并控制自己的主密钥来加密所有数据和通信，没有任何其他人可以控制（用户控制的加密）；
      </li>
      <li class="graf graf--li graf-after--li" id="82ba">
       <strong class="markup--strong markup--li-strong">
        验证权
       </strong>
       ：社会可以自由检查所有使用的软件的来源，并可以出于任何目的按照自己的意愿运行它（软件自由）；
      </li>
      <li class="graf graf--li graf-after--li" id="bced">
       <strong class="markup--strong markup--li-strong">
        被遗忘权
       </strong>
       ：服务提供商仅存储提供服务所需的最少个人数据。一旦不再需要数据，就会删除它（最小数据保留）；
      </li>
      <li class="graf graf--li graf-after--li" id="78a2">
       <strong class="markup--strong markup--li-strong">
        访问权
       </strong>
       ：在访问服务之前，任何人不得被歧视或被迫同意任何条款和条件（个人自由）。
      </li>
     </ul>
     <p class="graf graf--p graf-after--li" id="99a8">
      为了保护您自己的安全和隐私，您就需要自由和控制。如果没有自由，安全和隐私就需要对供应商的充分信任。然而，供应商并不总是将您的最大利益放在心上; 事实上，在许多情况下，供应商都有财务激励来侵犯您的利益，特别是在涉及隐私方面。问题是，使用专有软件很难拿到供应商值不值得信任的证据。
     </p>
     <p class="graf graf--p graf-after--p" id="05ec">
      使用自由软件产品，您可以控制自己的信任；您还可以验证您的自由软件供应商是否值得信赖。通过可重现的构建，您可以下载源代码并自行验证。
     </p>
     <p class="graf graf--p graf-after--p" id="9f43">
      最终，自由将导致更强的安全性和隐私性。这三个概念不仅相互关联，而且相互依存。随着您增加自由度，您可以提高安全性和隐私性，当降低自由度时，安全性和隐私就会被置于风险之中。
     </p>
