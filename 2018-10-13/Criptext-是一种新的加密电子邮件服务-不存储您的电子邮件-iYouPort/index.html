    Criptext 是一种新的加密电子邮件服务，不存储您的电子邮件

    McCaffrey Knowledge Node , 技术防身/自我保护方法 , 给行动者的护身符和武器 2018年10月13日 2019年9月8日 0 Minutes

    与其他加密电子邮件服务不同，Criptext 并不掌握用户的密钥，还是由用户自行生成并且存储在设备上。这就意味着，只有发件人和收件人双方能够知晓邮件内容

    Criptext 是一种新的加密电子邮件服务，“不会在其服务器中存储任何电子邮件”，而是“所有电子邮件都存储在您的设备上”。 对用户所发送的消息内容进行端对端加密 。

    Criptext 电子邮件于 2018 年8月作为测试版服务推出，它有一个相当有趣的原始故事，如果您有兴趣，可以 在这里 阅读。

    访问您的 Criptext 帐户只能通过使用它的 开源应用程序 来完成（适用于 macOS，Linux，Android和 iOS;即将推出 Windows 版）该服务现在可以免费使用。付费高级版可能会在未来推出，但免费版将继续推出。

    Criptext 电子邮件特点包括：

    • 端到端加密

    • “取消发送”电子邮件

    • 显示收件人读取

    • 开源应用程序

    Criptext 使用自己的唯一密钥加密每封电子邮件，因此即便密钥被泄露，您的收件箱也是安全的。此外，应用程序仅在您的设备上生成并存储加密密钥，而不是像 ProtonMail，在其安全页面上作为示例给出。

    引起我注意的是 Criptext 声称它不会将电子邮件存储在其服务器上。根据其 常见问题页面 ，有一些例外。

    例如，如果您已登录设备但未启动或未连接到 Internet，则电子邮件将存储在 Criptext 服务器上，直到您的设备能够接收电子邮件（已启动并连接到 Internet）。

    作为旁注，您还应该知道您需要登录至少一台设备，否则您将丢失任何可能收到的电子邮件。由于加密， 如果没有活动密钥，则无人可以访问传入的电子邮件。

    在“您存储关于我的哪些信息？”的回答中提到：“我们从不存储您的电子邮件，除非它是发送到非 Criptext 电子邮件地址的，在这种情况下，它会暂时存储在我们的服务器中。”

    此外，电子邮件附件（每封电子邮件限制为 20MB）在 Amazon Web Services 上加密存储。

    如果您想知道，是的，您可以使用 Criptext 接收非加密电子邮件，如果您愿意，也可以发送未加密的电子邮件。使用 Criptext 发送到其他 Criptext 电子邮件地址的电子邮件始终是加密的。

    您还可以使用 Criptext 将安全电子邮件发送到非 Criptext 电子邮件地址。在这种情况下，信息使用 Signal 协议进行加密，密钥使用 AES 进行加密，并且收件人会在 Web 浏览器中收到显示加密消息的链接。一旦打开，在到期之前您将有10天的时间阅读电子邮件。

    至于桌面应用程序，它包括双因素身份验证，使用高级过滤器搜索，撰写窗口的 WYSIWYG 编辑器，全部/未读取的视图切换、标签和自动签名功能。它缺乏使用第三方电子邮件客户端或多帐户支持的能力。◾️
