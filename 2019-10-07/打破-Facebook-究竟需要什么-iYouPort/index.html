    打破 Facebook 究竟需要什么？

    McCaffrey Newsletters , Opinion , 关于 Facebook , 技术 , 监视资本主义 2019年10月8日 2019年6月5日 1 Minute
    • 想要打破它，就需要分析以了解维系它的运转所依赖的东西究竟是什么

    这个世界与 Facebook 的关系已经从迷恋变为了被动攻击。

    人们对 Facebook 的情绪爱恨交织，经常能看到他们在 Facebook 上分享批评 Facebook 的帖子；以及愤怒驱动的注销但少卿便会回归。

    事实上， Facebook 已经做到了让人们很容易错误地接受它存在的严重问题，不论是数据管理、用户权利、隐私人权、透明度、还是认知控制。

    五月底， Facebook 的联合创始人克里斯休斯呼吁联邦贸易委员会与 Facebook 散伙。休斯指出， Facebook 是一个事实上的垄断企业。

    他绝不是第一个说出实话的人。

    经济学人是另一个先行者，值得注意的是，它还指出了 垄断的真实本质 ，而不仅仅针对 Facebook ：即 如果数据是新石油， BigTech 拥有的则是石油钻井平台 。

    Facebook 等数据驱动的产品首先获取数据，并用它来增强产品，以收集更多的数据。

    然而，正如 ZDNet 的 Larry Dignan 所指出的那样，联邦贸易委员会是否会将 Facebook 视为垄断是值得怀疑的。

    就算这种情况发生也需要一段时间；就算发生，散伙是否会起作用也是值得怀疑的 —— 正如互联网先驱 Jaron Lanier 所指出的那样，基于服务换取数据的商业模式中单独的打破总是会导致不对称。

    也就是说，就算 Facebook 本身被打破，基于相同模型的 Facebook 克隆体会接连出现，其结果可能没有太大区别。

    在演讲中 Lanier 反思了谷歌和 Facebook 等公司在数字文化基础上制造的 “ 全球性悲剧、令人震惊的荒谬错误 ” —— 以及我们如何改变这一窘境。他说： “ 如果两个人希望沟通我们无法建立一个社会，唯一可能的方式，它是由希望操纵它们的第三人资助的。 ”

    在这里看到 视频 ： https://www.ted.com/talks/jaron_lanier_how_we_need_to_remake_the_internet?utm_campaign=tedspread&utm_medium=referral&utm_source=tedcomshare

    要解决这个问题，需要结合使用技术、数据和社交工程学。

    数据文化：支付数据的价格

    关于人们对数据收集的态度的调查是有说服力的。

    受访者希望获得更多隐私，但是不愿意为此付费。他们中的很大一部分人甚至愿意提供更多数据以换取便捷的免费服务。这已经足够说明了目前世界上的数据主权和意识的状况。

    尽管在个人层面上进行分析应该是令人担忧的，但这本身就是最重要的。

    你的数据是为 Facebook 提供动力的石油。随着每一个滚动、点击、点赞、阅读，你一直不断地为它的垄断提供能源，你已经用你的手指投票支持它了 。虽然看起来是一种交易，但是， 即使您不使用 Facebook 、甚至没有拥有帐户， Facebook 也同样在尽力获取您的数据 。

    并且《 Facebook 给了全球当权者一个终极暴政蓝图 》

    Facebook 通过其实践积累的数据量意味着它在数据财富竞争和 AI 竞争中占据了极大的优势。

    这个世界正在越来越多地通过机器学习算法来运行，数据训练这些算法的重要性是至关重要的。由于研究人员需要数据来使算法发挥作用， Facebook 的数据库也吸引了大量人才。

    然而，这给我们带来了一个非常现实的问题。

    正如休斯指出的那样，在剑桥分析公司丑闻曝光后，对 Facebook 的强烈抗议已经愈演愈烈。

    关联《 心理操纵的秘密： ” 行为经济学 ”+PR= ？ 》

    但是， #deletefacebook 并未导致太多变化，因为缺乏可行的替代方案，意味着人们基本上只有两种选择：完全放弃社交网络，或继续使用 Facebook 。

    但除了 Facebook 之外真的别无选择吗？

    要回答这个问题，必须首先在功能方面分解 Facebook 。

    Facebook 确实汇集了强大的功能，它们都捆绑在一起，通过其后天的生态系统进一步扩展 —— WhatsApp 和 Instagram 。

    尽管很少有替代品能够以 Facebook 这样的无缝方式提供所有这些选择，但确实存在替代方案。

    WhatsApp 曾经是其中之一，可惜它现在被 Facebook 吞并了；其他消息应用程序也有联系人管理和对群组和渠道的支持，其中一些，如 Telegram 、 Mastodon 或 Diaspora ，也是开源的，它们使用户可以选择更多地控制他们的数据。

    但是，这些应用主要被已经脱离社交网络的人或边缘群体的人使用。那么，为什么它们没有赶上 Facebook ？

    碎片化是一个原因。

    如果您的某些联系人使用 A 应用程序，而其他人使用 B 应用程序，那么如何通过它们进行通信呢？

    但这已经不再是一个问题 —— 如果您从 Fediverse 、 Federation 、或 Activity Web 中发布，就可以做到跨平台沟通。

    详见我们曾经的详细介绍《如 何才能挑战 Facebook 并取得胜利？ 》

    因为这些应用程序构建在开放式通信协议之上。虽然每个应用程序都有自己的 UI / UX 和特权，但支持这些开放协议可确保基本功能适用于所有应用程序。

    这意味着您可以跨应用程序跟随人员和消息。你会觉得非常酷，的确是 —— 但是它还不够好。 Facebook 的核心功能不应该太难复制，许多替代社交网络应用程序已经做到了这一点。

    但是， 这些应用程序尚未成为主流的部分原因是，坦率地说，其中许多应用程序尚未做好准备 。

    它们的功能通常是有限的且不稳定的， UI / UX 并不理想，并且大量的应用程序基于旧的编程框架，这些框架难以跟上现代开发进度并吸引贡献者。例如，它们中的许多没有移动应用程序。

    然而，无论它们的实施质量如何，这些应用都有完全不同的理念。它们通常不由承担安装、配置和保持后端运行的商业实体托管。

    这意味着潜在用户可以做以下两件事之一：弄一个个人服务器；或者，找到运行他们想要使用的应用程序的服务器节点的人。第一种选择对绝大多数用户来说是不可行的。

    尽管像 Freedom Box 这样的努力有望使托管您自己的应用程序变得更容易，但它们尚未能提供无缝功能。

    第二种选择可能更为现实。各种各样的家庭、朋友和社区都可以参与进来，选择他们的应用程序并将其作为托管服务使用。

    但是，这两种选择都需要花钱。不只是支付运营费用；或许，更重要的是，支付获取所有权和控制数据的费用。

    你喜欢社会工程学吗？

    是的，需要一些社会工程来完成这项工作。即使有营销预算，但实现某些目标并非易事。总是有转换成本，除了必须提倡联合之外（一个人离开没有用，需要一群相关的人一起离开才行）， Facebook 也肯定不会让您轻松拿走数据并离开 。

    首先，直到最近，你甚至都没得选。现在 Facebook 中有数据导出功能，但 您获得的数据远非完整且可用于启动的 ，它的格式是专有的，你没法使用。而且缺少重要信息，最突出的是，没有点赞数据。

    你可能已经听到过心理学分析，也就是 “5 个点赞足以描述你 ” ，以及， Facebook 及其许多合作伙伴可以访问这些数据的事实。

    但是，生成数据的用户无权访问它。你在数据导出中得到的只是你点赞的东西 —— 而不是它意味着什么！而且这甚至会还不包含 Facebook 没有让你导出的所有其他信息。

    这些不仅仅是可以帮助你引导另一个应用程序的数据；并且，您或您授权的任何人都可能会使用这些数据来训练机器学习算法。

    不论有没有 GDPR ， Facebook 都能确保你拿不走这些数据 。

    因此，监管应该迫使 Facebook 运行用户只需按一下按钮就可以拿到所有信息。删除帐户及其所有数据也应该同样容易（目前的事实完全相反）。

    需要明确的是，在这个问题上，离开 Facebook 或任何其他平台都不会是最终的结果。但这将是实现这一目标的重要一步。它可以让用户随意选择如何使用他们的数据。

    在数据使用方面 Facebook 对美国和英国的政府关于透明度的呼吁表现出同样的蔑视。

    数据主权恰恰是由 Tim Berners Lee 倡导的应用程序开发框架 Solid 背后的愿景。 Berners Lee 在创业公司 Inrupt 以及来自开源和研究界的贡献者的帮助下，正在研究 Solid 。 在这里看到介绍《》

    Solid 正致力于开发 pods 。您可以将 pod 视为个人数据保险库，将所有数据保存在一个位置。从那里，您可以授权应用程序使用数据，以提供读取或写入权限。

    pod 可以托管在您自己的机器上，也可以托管在云端。

    听起来不错，但这很棘手。除了所有非技术原因外，还存在技术本身的障碍。

    如果实现，那就意味着应用程序必须从整个网络和云端的 pod 中获取数据才能工作。在分布式环境中查询是非常困难的。但 Solid 在那里有一张王牌：知识图谱 Knowledge Graph 。

    知识图谱是对可追溯到 20 年前的技术的重塑。前 Semantic Web 现在重新命名为 Linked Data 正在为此努力 ，这项技术实现了许多功能，包括联合查询。

    Linked Data stack （ RDF ， URI 和 SPARQL ）可以使任何数据在 Web 上可访问和查询。 Solid 基于这项技术，有效地旨在为每个用户构建个人知识图谱。

    毋庸置疑，这是非常雄心勃勃的，这可能也有助于解释为什么仍然没有适用于 Solid 的生产就绪的软件。

    SPARQL 不仅仅是一种查询语言。它也是一种协议，可用于跨 Web 上的许多端点执行联合查询。但从性能和可用性的角度来看，这远非微不足道。对于这类应用程序而言，使用 SPARQL 和 Linked Data stack 对于专业人士来说也具有挑战性。

    看起来 Solid 团队有心态和人才能使其发挥作用，而技术正在进行中。 Ruben Verborgh 、 Semantic Web 教授、研究员和 Inrupt 技术倡导者，与他的团队一起致力于利用工具来丰富 Linked Data stack ，以满足现代开发人员的需求。

    他们的工具基于 JavaScript 和 React 等框架，旨在使编程 Solid 成为一种无缝体验。

    Verborgh 和团队正在努力与开发者见面。他们不是期望开发人员切换到 Linked Data stack ，而是给他们提供在 Solid 上构建应用程序的工具。

    考虑到 React 在 Facebook 上的起源，这很聪明、务实、也有点讽刺。但它会起作用吗？

    Berners Lee 的杠杆作用可能有所帮助，但成功需要的不仅仅是好的意愿。

    2009 年， Berners Lee 在 TED 上呼吁观众支持他之前的项目 Linked Open Data ，这是关于将开放数据作为 Linked Data 的技术。今天，开放数据集可能正在增长，但这些数据集的可用方式是不同的。

    让 Solid 生产就绪并能够在其上构建应用程序，在很大程度上取决于对开发者的吸引。

    吸引开发者是一门艺术。也需要有很多钱。我们不知道 Solid 是否具备所有这些条件，它是一个很好的起点，但并不一定能保证成功。

    不过只要想想将 Fediverse 和 Solid 放在一起就可以开辟的可能性。这是社交网络变革的关键，赶走中间的独裁者，实现数据主权和一个全新的创新生态系统。

    祈祷 Solid 很快就会变得坚实吧。
